# HPA Expression Ingest Design

## Overview

Ingests two expression datasets from the Human Protein Atlas (HPA) v25.0:
- **IHC protein expression** — antibody-based protein detection across normal tissues
- **RNA expression** — RNA-seq nTPM values across normal tissues

Both are wired into `target_graph.yaml`, `pharos.yaml`, and `working.yaml` via `HPAProteinExpressionAdapter` and `HPARnaExpressionAdapter` in `src/input_adapters/hpa/hpa_expression.py`.

---

## Input Files

| File | URL | Purpose |
|------|-----|---------|
| `normal_ihc_data.tsv.zip` | `https://www.proteinatlas.org/download/tsv/normal_ihc_data.tsv.zip` | IHC protein expression |
| `rna_tissue_hpa.tsv.zip` | `https://www.proteinatlas.org/download/tsv/rna_tissue_hpa.tsv.zip` | RNA expression |
| `hpa_version.tsv` | generated by Snakemake | Version and release date |
| `src/input_adapters/shared/manual_uberon_map.tsv` | committed to repo | Tissue/cell-type → UBERON/CL ID map |

Downloaded to `input_files/auto/hpa/` via `rule download_hpa` in `workflows/pharos.Snakefile`.

### Version strategy
- `version`: scraped from `https://www.proteinatlas.org/about/download` HTML (regex `version \d+\.\d+`)
- `version_date`: parsed from `Last-Modified` header on the zip download
- `download_date`: computed in the adapter from `os.path.getmtime()`

## Model Mapping

Both adapters emit:
- `Protein` nodes keyed by `EquivalentId(ensg, Prefix.ENSEMBL)` — resolved to UniProt by `tg_proteins` resolver
- `Tissue` nodes keyed by UBERON ID (from map) or `EquivalentId(name, Prefix.Name)` as fallback — normalized by `TissueResolver`
- `ProteinTissueExpressionEdge` with one or more `ExpressionDetail` items

### HPAProteinExpressionAdapter
- One edge per gene/tissue (canonical `Tissue` name, not `IHC tissue name`)
- One `ExpressionDetail` per cell type on that edge:
  - `source = "HPA Protein"`
  - `qual_value` = Level string (stored as-is including non-ordinal values)
  - `evidence` = Reliability
  - `cell_type` = Cell type string
  - `source_id` = ENSG gene ID
  - `source_tissue_id` = map lookup by `"tissue - cell_type"` first, then `"tissue"`
  - `expressed` = `level != "Not detected"` (only ordinal-level rows are kept)
  - `source_rank` = tissue-level normalized rank (same value for all cell types on a tissue)
- `Protein.calculated_properties["hpa_ihc_tau"]` = tau across tissues

**Tau for IHC:** Ordinal values (0–3) per the `qualMap` from the original TCRD code. Tau is computed from the max ordinal per tissue, then across all tissues. Non-ordinal levels are excluded from tau computation.

### HPARnaExpressionAdapter
- One edge per gene/tissue, one `ExpressionDetail`:
  - `source = "HPA RNA"`
  - `number_value` = nTPM
  - `source_id` = ENSG gene ID
  - `source_tissue_id` = map lookup by tissue name
  - `expressed` = `nTPM > 0`
  - `source_rank` = normalized rank
- `Protein.calculated_properties["hpa_rna_tau"]` = tau across tissues

## UBERON / CL ID Mapping

### Tissues
Tissue names are mapped via `src/input_adapters/shared/manual_uberon_map.tsv` (also used by HPM). IHC uses Title Case (`"Adipose tissue"`); RNA uses lowercase (`"adipose tissue"`). Both are lowercased for lookup.

Plain tissue name entries for all 48 HPA tissues have been added to the map (e.g. `liver → UBERON:0002107`, `testis → UBERON:0000473`). All tissues now resolve to UBERON IDs rather than `Name:` fallbacks.

### Cell types
The map has entries for `"tissue - cell type"` combos (e.g. `"liver - hepatocytes" → UBERON:0002107`). These are used for `ExpressionDetail.uberon_id`. The 76 canonical (cell type, tissue) combos in `normal_ihc_cell_types.tsv` are a candidate for a future CL ID mapping.

## Differences from Old TCRD Implementation

The old Airflow DAG (`buildExpression.py`) differs in a few areas:

| Aspect | Old TCRD | New adapter |
|--------|----------|-------------|
| Reliability filter | Drops `Uncertain` rows | Same ✓ |
| Non-ordinal levels | Skips entirely | Same ✓ |
| IHC tau input | One value per gene/tissue/celltype row | Max ordinal per tissue, then across tissues |
| Edge structure | Flat (one row per gene/tissue/celltype) | Grouped (one edge per gene/tissue, details list per cell type) |
| Source strings | `'HPA RNA'`, `'HPA Protein'` | Same ✓ |
| Protein ID resolution | Internal MySQL lookup | `tg_proteins` resolver via ENSG |

## Shared Infrastructure

Both adapters inherit from `ExpressionAdapterBase` (`src/input_adapters/shared/expression_adapter_base.py`), which also backs `HPMExpressionAdapter`. Shared methods:
- `_load_version_info`
- `_load_uberon_map`
- `_tissue_id`
- `_compute_tau`
- `_normalized_rank`

The UBERON map (`src/input_adapters/shared/manual_uberon_map.tsv`) is committed to the repo and shared across HPM and HPA adapters. It was previously located at `input_files/manual/hpm/manual_uberon_map.tsv` (not committed).

### RecordMerger dict handling
`Protein.calculated_properties` is a `Dict[str, float]`. When multiple adapters (IHC and RNA) both emit a `Protein` with different keys in `calculated_properties`, `RecordMerger.merge_records` now merges the dicts rather than replacing them — `KeepFirst` preserves existing keys, `KeepLast` lets new keys win. This applies generically to any dict-valued field.
