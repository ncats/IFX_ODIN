## Scope
Ingests Reactome pathway data (human-only) into the graph from the downloads above.

## Data pulled from Reactome

### 1) Pathway list (from `ReactomePathways.gmt.zip`)
- GMT row structure: `name \t description \t member1 \t member2 ...`
- The adapter extracts the Reactome stable ID `R-HSA-####` from the name only.
- Only human pathways are kept (IDs starting with `R-HSA-`).

### 2) Pathway hierarchy (from `ReactomePathwaysRelation.txt`)
- TSV with `parent_pathway_id \t child_pathway_id`.
- Only rows where both IDs are human (`R-HSA-`) are kept.

### 3) Protein-to-pathway links (from `UniProt2Reactome_All_Levels.txt`)
- TSV contains `UniProtKB` protein ID, pathway ID, and species.
- Only rows where pathway ID is `R-HSA-` and species contains `Homo sapiens` (or `9606`) are kept.

## Mapping to the data model

### Nodes
- **Pathway** (`src/models/pathway.py`)
  - `id`: Reactome stable ID (e.g., `R-HSA-199420`)
  - `source_id`: same as `id`
  - `type`: `Reactome`
  - `name`: pathway name from GMT
  - `url`: `https://reactome.org/content/detail/<id>`

### Edges
- **PathwayParentEdge** (`src/models/pathway.py`)
  - `start_node`: parent `Pathway` (Reactome ID)
  - `end_node`: child `Pathway` (Reactome ID)
  - `source`: `Reactome`

- **ProteinPathwayRelationship** (`src/models/pathway.py`)
  - `start_node`: `Protein` (ID is `UniProtKB:<uniprot_id>`)
  - `end_node`: `Pathway` (Reactome ID)
  - `source`: `Reactome`

## Adapters

- `ReactomePathwayAdapter`
  - Input: `ReactomePathways.gmt.zip`
  - Output: `Pathway` nodes

- `ReactomePathwayParentEdgeAdapter`
  - Input: `ReactomePathwaysRelation.txt`
  - Output: `PathwayParentEdge` edges

- `ReactomeProteinPathwayEdgeAdapter`
  - Input: `UniProt2Reactome_All_Levels.txt`
  - Output: `ProteinPathwayRelationship` edges

## Notes / Assumptions
- The GMT is read directly from the zip; no unzip step required in Snakemake.
- Human filtering is done in the adapters, not during download.
- Reactome version is pulled from `https://reactome.org/ContentService/data/database/version`.
- Reactome version_date is derived from the `Last-Modified` header of
  `https://reactome.org/download/current/ReactomePathways.gmt.zip`.
- A `reactome_version.tsv` file is generated by Snakemake and passed to adapters
  to populate `DatasourceVersionInfo.version` and `version_date`.
