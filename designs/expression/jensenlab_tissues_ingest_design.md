# JensenLab TISSUES Expression Ingest Design

## Overview

Ingests the JensenLab TISSUES integrated confidence scores — an aggregate across RNA-seq experiments,
IHC data, and text mining — for human protein-coding genes across human-relevant BTO/CLDB tissues.

Wired into `target_graph.yaml`, `pharos.yaml`, and `working.yaml` via
`JensenLabTissuesExpressionAdapter` in `src/input_adapters/jensenlab/tissues_expression.py`.

---

## Input Files

| File | URL | Purpose |
|------|-----|---------|
| `human_tissue_integrated_full.tsv` | `https://download.jensenlab.org/human_tissue_integrated_full.tsv` | Integrated expression confidence scores |
| `tissues_version.tsv` | generated by Snakemake | Version date from HTTP `Last-Modified` header |

Downloaded to `input_files/auto/jensenlab/` via `rule download_jensenlab_tissues` in
`workflows/pharos.Snakefile`.

### Version strategy
- JensenLab regenerates these files on a recurring schedule.
- `version_date`: parsed from the `Last-Modified` HTTP header on the data file download, written to
  `tissues_version.tsv` by Snakemake.
- `download_date`: computed in the adapter from `os.path.getmtime()`.

## Data Profiling

The file has no header row. Tab-delimited, 5 columns:

| Column | Content |
|--------|---------|
| `gene_id` | Ensembl protein ID (`ENSP…`) or gene symbol (miRNA, lncRNA, rRNA, etc.) |
| `gene_name` | Gene symbol |
| `ontology_id` | BTO or CLDB tissue/cell-line ontology ID |
| `tissue` | BTO tissue name |
| `confidence` | Aggregate confidence score (0.001 – 5.0) |

Gene IDs filtered to `ENSP`, `ENSG`, or `ENST` prefix (human Ensembl IDs). `startswith("ENS")` alone
is insufficient — mouse IDs start `ENSMUSP`/`ENSMUSG`/`ENSMUST`. All 9,179,748 ENS-prefixed rows
in this file are `ENSP`, confirming the file is human-only in practice.

Non-ENSP/ENSG/ENST rows (miRNAs like `hsa-miR-…`, rRNAs like `18S_rRNA`, lncRNA aliases) are skipped.

| Total rows | 9,179,748 |
| Unique tissue IDs (raw) | 6,841 |
| Score range | 0.001 – 5.0 |
| Scores > 0 | 100% |
| Tissues per gene (median) | 357 |

**Ontology ID prefixes (after gene filter):**

The raw BTO/CLDB ID set spans all organisms (plants, bacteria, cell culture media, non-human cell
lines). The adapter pre-filters to human-relevant IDs only via FMA xref — see **Human Tissue
Filtering** below.

## Human Tissue Filtering

`JensenLabTissuesExpressionAdapter` accepts an optional `obo_file_path` parameter pointing to
`uberon.obo`. When provided, it builds an allowlist of BTO and CLDB IDs at startup by scanning every
term in the OBO for xrefs — but only keeping terms that also carry an **FMA xref**.

FMA (Foundational Model of Anatomy) is a human-specific ontology. UBERON/CL terms with an FMA xref
are human-grounded by definition. This is the same filter used by the old TCRD `getOntologyMap()`
SQL query (`WHERE uberon_xref.xref LIKE 'BTO:%' AND fma_xref IS NOT NULL`).

Effect:
- BTO IDs for plant, bacterial, fungal, and non-human animal tissues are excluded
- CLDB IDs for non-human cell lines are excluded
- Human cell lines with FMA-grounded CL equivalents (e.g. `CL:0000842` mononuclear leukocyte →
  `BTO:0000878`) are retained

Human gene filtering is handled separately by the `startswith("ENS")` check on `gene_id` — all
ENSP IDs in this file are human, equivalent to the old `humanEntityMap` lookup.

## Model Mapping

The adapter emits:
- `Tissue` nodes keyed by BTO or CLDB ID — resolved to their canonical UBERON/CL ID by
  `TissueResolver` (both `BTO` and `CLDB` are in `valid_ontologies`)
- `ProteinTissueExpressionEdge` with one `ExpressionDetail` per gene/tissue — the edge's
  `start_node` carries the ENSP/ENSG ID, resolved to UniProt by `tg_proteins` / `tcrd_targets`
  (same pattern as `TotalPMScoreAdapter`). No separate `Protein` node is emitted since there is no
  additional protein-level data to contribute.

### ExpressionDetail fields
| Field | Value |
|-------|-------|
| `source` | `"JensenLab"` |
| `tissue` | BTO tissue name string |
| `source_id` | ENSP or ENSG ID from the file |
| `source_tissue_id` | BTO ID (e.g. `BTO:0000567`); `TissueResolver` maps it to UBERON on the node |
| `number_value` | confidence score (float) |
| `expressed` | `confidence > 0` (always `True` in practice — min observed score is 0.001) |
| `source_rank` | normalized rank across tissues for this gene |

### Protein.calculated_properties
None. Tau is not computed for JensenLab TISSUES because the BTO tissue set is ontology-driven and
multi-granular (e.g. "liver", "hepatocytes", and "liver parenchymal cell" are all separate entries),
and cell lines (CLDB) are mixed with anatomical tissues. A flat tau across this non-curated set
would not be meaningful. The other expression sources (HPM, HPA, GTEx) all use curated,
non-overlapping tissue panels where tau is well-defined.

---

## UBERON / BTO / CLDB Mapping

Unlike HPA and HPM, this adapter does **not** use `manual_uberon_map.tsv`. BTO and CLDB IDs are
emitted directly as `Tissue` node IDs and resolved by `TissueResolver` via the UBERON OBO xref
reverse map. Both `BTO` and `CLDB` are listed in `valid_ontologies` in all three YAML configs.

`TissueResolver` now indexes all terms in `uberon.obo` (not just `UBERON:` prefixed) and builds a
reverse map from any xref to its canonical OBO term — so BTO IDs resolve to their UBERON or CL
equivalent, and CLDB IDs resolve similarly where mappings exist.

## Differences from Old TCRD Implementation

| Aspect | Old TCRD | New adapter |
|--------|----------|-------------|
| Gene ID type | Mapped gene symbols via `humanEntityMap` | Filter to `ENSP`/`ENSG`/`ENST` prefix |
| Human gene filtering | Explicit `humanEntityMap` lookup | `ENSP`/`ENSG`/`ENST` prefix check — excludes mouse (`ENSMUSP` etc.) and non-coding aliases |
| Human tissue filtering | `getOntologyMap()` — BTO IDs from UBERON/CL terms with FMA xref | Same — `_load_valid_tissue_ids()` scans OBO for FMA-grounded BTO/CLDB IDs |
| Tissue ID | Manual map first, then ontology map; skip if unresolved | BTO/CLDB ID emitted; `TissueResolver` maps to canonical UBERON/CL |
| Tau | Not computed | Not computed — BTO tissue set is non-curated, multi-granular, mixed with cell lines |
| Rank | Not computed | `source_rank` via `ExpressionAdapterBase._normalized_rank()` |
| Expressed flag | `confidence > 0` | Same ✓ |
| Source string | `'JensenLab'` | Same ✓ |

## Shared Infrastructure

Uses `ExpressionAdapterBase._normalized_rank()` as
static calls without inheriting the full base class (no version file or UBERON map needed).

The same ENSP → UniProt resolution path used by `TotalPMScoreAdapter` applies here.
